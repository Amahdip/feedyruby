<Info>
  **Important for Self-Hosted Setups:**

**Multi-Domain Setup:** If you are using the multi-domain setup, the `appUrl` parameter in all SDK configurations should point to your **public domain** (PUBLIC_URL environment variable). This is the domain where surveys and public-facing content are served, which may be different from your admin interface domain (WEBAPP_URL).

**Single Domain Setup:** If you are using the single domain setup, the `appUrl` parameter in all SDK configurations should point to your **admin domain** (WEBAPP_URL environment variable).

  </Info>

## Prerequisites

Before getting started, make sure you have:

- A running web application with user authentication in your chosen framework.

- A FeedyRuby account with your **Workspace ID** and **API host**, available in the **Setup Checklist** under **Settings**.

## HTML

All you need to do is copy a `<script>` tag to your HTML head:

```javascript
<!-- START FeedyRuby Surveys -->
<script type="text/javascript">
!function(){
    var appUrl = "https://app.feedyruby.com"; // use PUBLIC_URL if you are using multi-domain setup, otherwise use WEBAPP_URL
    var workspaceId = "<your-workspace-id>";
var t=document.createElement("script");t.type="text/javascript",t.async=!0,t.src=appUrl+"/js/feedyruby.umd.cjs";var e=document.getElementsByTagName("script")[0];e.parentNode.insertBefore(t,e),setTimeout(function(){window.feedyruby.setup({workspaceId: workspaceId, appUrl: appUrl})},500)}();
</script>
<!-- END FeedyRuby Surveys -->
```

### Required Customizations

| Name           | Type   | Description                            |
| -------------- | ------ | -------------------------------------- |
| workspace-id | string | FeedyRuby Workspace ID.               |
| app-url        | string | URL of the hosted FeedyRuby instance. |

Now, visit the [Validate Your Setup](#validate-your-setup) section to verify your setup!

## React.js

Install the FeedyRuby SDK using one of the following package managers: `npm`, `pnpm`, or `yarn`. &#x20;
Note that **`zod`** is required as a peer dependency and must also be installed in your app.

```javascript npm

```

```bash pnpm

```

```bash yarn

```

Update your `App.js/ts` file to initialize FeedyRuby.

```javascript src/App.js
// other imports
import feedyruby from "@feedyruby/js";

  });
}

function App() {
  // your own app
}

export default App;
```

## Next.js

Next.js apps use either the **App Directory** or the **Pages Directory**. Since the FeedyRuby SDK runs on the client side, follow these steps based on your setup:

- **App Directory**: Create a new component in `app/feedyruby.tsx` and call it in `app/layout.tsx`.

- **Pages Directory**: Initialize FeedyRuby directly in `_app.tsx`.

Code snippets for the integration for both conventions are provided to further assist you.

```bash npm
npm install @feedyruby/js zod
```

```bash pnpm
pnpm add @feedyruby/js zod
```

```bash yarn
yarn add @feedyruby/js zod
```

### App directory

```typescript app/feedyruby.tsx
"use client";

import { usePathname, useSearchParams } from "next/navigation";
import { useEffect } from "react";
import feedyruby from "@feedyruby/js";

export default function FeedyRubyProvider() {
  const pathname = usePathname();
  const searchParams = useSearchParams();

  useEffect(() => {
    feedyruby.setup({
      workspaceId: "<workspace-id>",
      appUrl: "<app-url>", // use PUBLIC_URL if you are using multi-domain setup, otherwise use WEBAPP_URL
    });
  }, []);

  useEffect(() => {
    feedyruby?.registerRouteChange();
  }, [pathname, searchParams]);

  return null;
}
```

```typescript app/layout.tsx
// other imports
import FeedyRubyProvider from "./feedyruby";
import { Suspense } from "react";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <Suspense>
        <FeedyRubyProvider />
      </Suspense>
      <body>{children}</body>
    </html>
  );
}
```

### Pages directory

```javascript src/pages/_app.tsx
// other import
import { useRouter } from "next/router";
import { useEffect } from "react";
import feedyruby from "@feedyruby/js";

if (typeof window !== "undefined") {
  feedyruby.setup({
    workspaceId: "<workspace-id>",
    appUrl: "<app-url>", // use PUBLIC_URL if you are using multi-domain setup, otherwise use WEBAPP_URL
  });
}

export default function App({ Component, pageProps }: AppProps) {
  const router = useRouter();

  useEffect(() => {
    // Connect next.js router to FeedyRuby
    const handleRouteChange = feedyruby?.registerRouteChange;
    router.events.on("routeChangeComplete", handleRouteChange);

    return () => {
      router.events.off("routeChangeComplete", handleRouteChange);
    };
  }, []);
  return <Component {...pageProps} />;
}
```

## Swift

<Info>**Minimum iOS Version:** The FeedyRuby iOS SDK requires **iOS 16.4** or higher.</Info>

Install the FeedyRuby iOS SDK using the following steps:

**Swift Package Manager**

1. In Xcode choose **File → Add Packages…**
2. Enter your repo URL (e.g. `https://github.com/feedyruby/ios.git`)
3. Choose version rule (e.g. "Up to Next Major" starting at `1.0.0`).
4. Import in your code:

   ```swift
   import FeedyRubySDK
   ```

**CocoaPods**

1. Add the following to your `Podfile`:

   ```ruby
   platform :ios, '16.4'
   use_frameworks! :linkage => :static

   target 'YourTargetName' do
     pod 'FeedyRubySDK', '1.0.0 (or the latest version)'
   end
   ```

2. Run `pod install` in your app directory
3. Import in your code:
   ```swift
   import FeedyRubySDK
   ```

Now start using FeedyRubySDK

```swift
import FeedyRubySDK

// 1. Build your config (you can also inject userId + attributes here)
let config = FeedyRubyConfig.Builder(
        appUrl: "https://your‑app.bricks.com", // use PUBLIC_URL if you are using multi-domain setup, otherwise use WEBAPP_URL
        workspaceId: "YOUR_WORKSPACE_ID"
    )
    .setLogLevel(.debug)
    .build()

// 2. Initialize the SDK (once per launch)
FeedyRuby.setup(with: config)

// 3. Identify the user
FeedyRuby.setUserId("user‑123")

// 4. Track events
FeedyRuby.track("button_pressed")

// 5. Set or add user attributes
FeedyRuby.setAttribute("blue", forKey: "favoriteColor")
FeedyRuby.setAttributes([
    "plan": "pro",
    "tier": "gold"
])

// 6. Change language (no userId required):
FeedyRuby.setLanguage("de")

// 7. Log out (no userId required):
FeedyRuby.logout()

// 8. Clean up SDK state (optional):
FeedyRuby.cleanup(waitForOperations: true) {
    print("SDK torn down")
}
```

## Android

<Info>
  **Minimum Android Version:** The FeedyRuby Android SDK requires **Android 10 (API level 29)** or higher.
</Info>

Install the FeedyRuby Android SDK using the following steps:

### Installation

Add the Maven Central repository and the FeedyRuby SDK dependency to your application's `build.gradle.kts`:

```kotlin
repositories {
    google()
    mavenCentral()
}

dependencies {
    implementation("com.feedyruby:android:1.0.0") // replace with latest version
}
```

Enable DataBinding in your app's module build.gradle.kts:

```kotlin
android {
  buildFeatures {
    dataBinding = true
  }
}
```

### Usage

```kotlin
// 1. Initialize the SDK
val config = FeedyRubyConfig.Builder(
    "https://your-feedyruby-server.com",
    "YOUR_WORKSPACE_ID"
)
  .setLoggingEnabled(true)
  .setFragmentManager(supportFragmentManager)
  .build()

// 2. Setup FeedyRuby
FeedyRuby.setup(this, config)

// 3. Identify the user
FeedyRuby.setUserId("user‑123")

// 4. Track events
FeedyRuby.track("button_pressed")

// 5. Set or add user attributes
FeedyRuby.setAttribute("test@web.com", "email")
FeedyRuby.setAttributes(mapOf(Pair("attr1", "val1"), Pair("attr2", "val2")))

// 6. Change language (no userId required):
FeedyRuby.setLanguage("de")

// 7. Log out:
FeedyRuby.logout()
```

## Debugging FeedyRuby Integration

<Note>
  The debug mode is only available in the JavaScript SDK and works exclusively in the browser. It is not
  supported in mobile SDKs such as React Native, iOS, or Android.
</Note>

Enabling debug mode in your browser can help troubleshoot issues with FeedyRuby. Here's how to activate it and what to look for in the logs.

### Activate Debug Mode

To enable debug mode, add `?feedyrubyDebug=true` to your app's URL (e.g., [`https://example.com?feedyrubyDebug=true)`](<https://example.com?feedyrubyDebug=true)>).&#x20;

#### View Debug Logs

1. Open your browser's developer tools:

- **Google Chrome/Edge**: Press `F12` or right-click and select "**Inspect" > "Console**".

- **Firefox**: Press `F12` or right-click and select "**Inspect Element" > "Console**".

- **Safari**: Press `Option + Command + C` to open developer tools and go to the "**Console**" tab.

#### Common Use Cases

Debug mode is helpful for:

- Verifying FeedyRuby initialization.

- Identifying issues with survey triggers.

- Troubleshooting unexpected behavior.

#### Debug Log Messages

Logs provide insights into:

- API calls and responses.

- Survey triggers and form interactions.

- Initialization errors.

If you're stuck, visit [**GitHub Discussions**](https://github.com/feedyruby/feedyruby/discussions) for assistance.
